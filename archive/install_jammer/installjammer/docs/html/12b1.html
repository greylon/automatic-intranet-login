1.2b1
 		   

1.2b1 (May 28th, 2007)


This release contains many new enhancements and bug fixes.  The installkits have been recompiled with new features, which is the biggest reason for a minor revision bump instead of just a patch level.



Major Enhancements

  • Installkit binaries have been recompiled with new features.  FreeBSD remains to be compiled but will be completed before the final 1.2 release.
  • Added the ability to copy-and-paste panes, actions and conditions through right-click and the Edit menu. [Req #1664096]
  • Added new Install Password property to encrypt installers with a password. [Req #1557456]
  • Added support for mounting and handling zip files from within an installer.
  • Added a new section to specify which languages are supported by an installer instead of just being all or none. [Req #1661645]
  • Added new Unzip File action to unzip standard .zip files during installation.



Major Bug Fixes

  • Fixed a bug where the uninstaller would not remember the settings from its installer. [Bug #1676588]
  • Fixed a bug where loading a project with a lot of files could take minutes to load. [Req #1665160]
  • Fixed a bug where newly-created installs would inherit the Application ID of the last install opened. [Bug #1689940]



Minor Enhancements

  • Quick builds should be a lot faster now, especially on large projects with a lot of files.
  • Added new FetchURL API call to fetch a web URL.
  • Added debug output on API calls to show when they are called and with what arguments.
  • Added debug output on several actions that didn't have any.
  • Added an option to preserve file permissions for files on non-Windows platforms.
  • Reordered the check buttons on UNIX file permissions tabs to reflect the standard order.
  • Only languages included are stored within an installer which decreases size a little.
  • Automatically include the TWAPI extension Windows if any code uses it.
  • Added a new Require JDK property to the Locate Java Runtime action to require a full JDK instead of just a JRE.  This will set a new virtual text: <%JavacExecutable%> if the JDK is found. [Req #1690206]
  • Added a new Foreground property to the Add Widget action to specify the text color of most widgets.
  • Added a new label frame widget type to the Add Widget action.
  • Added a text widget type to the Add Widget action.
  • Added a new readonly state to the Modify Widget action that can be applied to entry and text widgets.
  • Changed the popup when testing a modified project.  Now, instead of just asking whether you want to save and build, you have the option to quick build or do a full build.
  • Only show the aliases of actions when selecting from a list instead of showing object IDs that don't mean anything to the user. [Req #1664071]
  • Added new Watch Progressive Output property to the Execute External Program action to watch the output of a program for special cues to InstallJammer.  This allows external programs to change the values of virtual text in InstallJammer when running for a long time so they can update messages and progress bars, etc...



Minor Bug Fixes

  • Fixed a bug where clicking twice on a grouping in the user interface tree (like attempting to rename) would cause an error [Bug #1665811]
  • Fixed a bug where selecting multiple panes and then adding a new action would cause an error.  Actions are now added to the last pane selected.
  • Fixed a bug where substituting text for eval would break if the virtual text had multiple words.
  • Fixed a minor bug in the text edit window when cancelling an edit.
  • Fixed a bug in the PromptForFile API that would cause an error if the user hit cancel.
  • Fixed a bug in the Populate Components action that would populate components that were marked inactive.
  • Fixed a bug where adding multiple files to a file group would cause an error. [Bug #1702000]
  • Some fixes were made to attempt better cleanup of stopped builds. [Bug #1664102]
  • Fixed a bug where UNIX file permissions were not set properly based on the mask. [Bug #1661644]
  • Fixed a bug in the Launch File action that would cause backslashes to be substituted.
  • Removed the requirement in the Install Windows Shortcut action that the target file must exist to create the shortcut.  This will allow for creating shortcuts that point to launching programs. [Req #1682449]
  • Fixed the Windows Registry actions to propertly substitute virtual text. [Bug #1685186]
  • Fixed a bug where a destroyed window can still be in the parent window list and cause an error. [Bug #1672859]
  • The Ask Yes or No condition now includes the Console Ask Yes or No action and will use that when installing in console mode. [Bug #1713635]
  • Fixed a bug where checking for the existence of a registry key without a value would always return false.
  • Cleanup failed previews better.  Currently, if a preview fails, you can't preview that window again because of further errors. [Bug #1546595]
  • Fixed a bug where a small, blank window would popup during silent installs and uninstalls on Windows.
  • Fixed a bug in the Execute External Program action where the Status Virtual Text was not being accurately set with the exit code from the external program.
  • Fixed a bug where deleting a condition while still editing a property could cause an error in the condition editor. [Bug #1708594]
  • Fixed a bug where message boxes could popup as non-transient windows.
  • Added missing icons to installers. [Bug #1707437]
  • Fixed several smaller bugs on Windows 98. [Bug #1684175]
  • The choose directory dialog no longer has hard-coded English strings.  I still need to get translations for the other languages though. [Bug #1512658]
